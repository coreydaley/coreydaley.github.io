# Sprint 001: Co-located Page Bundles with Year/Month Organization

## Overview

Every blog post will become a Hugo **leaf bundle** organized by year and month:

```
content/posts/2026/02/welcome-to-my-new-blog/
├── index.md                    ← the post
├── robot-writing-code.webp     ← hero image
└── thumbs/
    └── robot-writing-code.webp ← thumbnail
```

This makes the post–image relationship explicit to both humans and AI agents: navigate
to a post's directory and find everything related to that post in one place. No more
cross-referencing filenames between `content/posts/` and `static/images/posts/`.

Old URLs (`/posts/welcome-to-my-new-blog/`) redirect to new URLs
(`/posts/2026/02/welcome-to-my-new-blog/`) via Hugo `aliases` in frontmatter, which
generate static HTML redirect pages. This is the correct GitHub Pages-compatible
redirect mechanism — no `.htaccess` required.

Hugo's default URL derivation from directory structure already produces the right URL
from `content/posts/YYYY/MM/slug/index.md` → `/posts/YYYY/MM/slug/`. No `[permalinks]`
config change is needed or wanted (adding one would create drift risk between folder
paths and frontmatter dates).

All theme templates are updated with **dual support**: absolute `image` paths (starting
with `/`) continue to work exactly as before; relative paths (e.g., `"hero.webp"`) are
resolved as Hugo page resources. This allows incremental migration without breaking any
post that hasn't been converted yet.

## Use Cases

1. **Author adds an image to a post**: Drop the WebP into the post's bundle directory,
   run `./scripts/optimize-images.sh`, reference by filename in the shortcode. No
   hunting through a shared pool.

2. **AI agent creates a new post**: Run `hugo new posts/2026/03/my-topic/index.md`,
   place generated image in `content/posts/2026/03/my-topic/`, run optimization.
   The agent can verify co-location with `ls content/posts/2026/03/my-topic/`.

3. **Reader follows a bookmarked old URL**: `/posts/old-slug/` serves a static HTML
   redirect (generated by Hugo from `aliases`) to `/posts/YYYY/MM/old-slug/`.

4. **Developer browses the repo**: Directory tree communicates post structure and image
   ownership immediately, without opening any file.

5. **AI agent audits post images**: `find content/posts -name "*.webp"` returns all post
   images alongside their posts. No inference required.

### URL Changes

Hugo derives URLs from the filesystem path for leaf bundles. No permalink config change.

| Old URL | New URL | Mechanism |
|---------|---------|-----------|
| `/posts/welcome-to-my-new-blog/` | `/posts/2026/02/welcome-to-my-new-blog/` | `aliases` in frontmatter |
| `/posts/attractor-dot-pipeline-orchestration/` | `/posts/2026/03/attractor-dot-pipeline-orchestration/` | `aliases` in frontmatter |
| All 20 posts... | `/posts/YYYY/MM/slug/` | `aliases` in frontmatter |

### Frontmatter Changes (per post)

```toml
# BEFORE
image = "/images/posts/robot-writing-code.webp"

# AFTER — add aliases array, change image to relative filename
aliases = ["/posts/welcome-to-my-new-blog/"]
image = "robot-writing-code.webp"
```

### Shortcode Changes (per post)

```
# BEFORE
{{< figure-float src="/images/posts/robot-writing-code.webp" alt="..." >}}
{{< figure-float src="/images/posts/robot-writing-code.png" alt="..." >}}

# AFTER — filename only, always .webp
{{< figure-float src="robot-writing-code.webp" alt="..." >}}
```

Note: All `src=` references must use `.webp` extension (not `.png`). PNG originals are
preserved locally but stripped from the deployed site by CI and must not be the src value.

### Dual-Support Theme Logic

All four templates are updated to handle both absolute and relative `image` values:

**`layouts/shortcodes/figure-float.html` and `figure-block.html`**:
```gotemplate
{{- $src := .Get "src" -}}
{{- if not (hasPrefix $src "/") -}}
  {{- $res := .Page.Resources.GetMatch $src -}}
  {{- if $res -}}{{- $src = $res.RelPermalink -}}{{- end -}}
{{- end -}}
{{- $webp := $src | replaceRE `\.(png|jpe?g)$` ".webp" -}}
```

**`themes/coreydaley-dev/layouts/_default/baseof.html`** — og:image/twitter:image:
```gotemplate
{{- $image := .Params.image | default .Params.avatar | default .Site.Params.avatar -}}
{{- if $image -}}
  {{- if hasPrefix $image "/" -}}
    {{- $image = $image | absURL -}}
  {{- else -}}
    {{- $res := .Resources.GetMatch $image -}}
    {{- if $res -}}{{- $image = $res.Permalink -}}{{- end -}}
  {{- end -}}
{{- end -}}
```

**`themes/coreydaley-dev/layouts/partials/post-list.html`** — thumbnail:
```gotemplate
{{- $imgParam := $post.Params.image -}}
{{- $img := "" -}}
{{- $thumb := "" -}}
{{- if hasPrefix $imgParam "/" -}}
  {{/* Legacy absolute path */}}
  {{- $img = $imgParam | replaceRE `\.(png|jpe?g)$` ".webp" -}}
  {{- $thumb = $img | replaceRE `^(/images/posts/)` "${1}thumbs/" -}}
{{- else -}}
  {{/* Bundle-relative resource */}}
  {{- with $post.Resources.GetMatch $imgParam -}}{{- $img = .RelPermalink -}}{{- end -}}
  {{- with $post.Resources.GetMatch (printf "thumbs/%s" $imgParam) -}}
    {{- $thumb = .RelPermalink -}}
  {{- else -}}
    {{- $thumb = $img -}}
  {{- end -}}
{{- end -}}
```

## Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| og:image produces wrong URL for relative paths | Medium | High | Dual-support theme logic in Phase 1; verify with a social preview tool after pilot |
| Post list thumbnails break during migration | Medium | Medium | Dual-support in Phase 1 before any content moves |
| Hugo leaf bundle strips images from resources (Hugo bug/config) | Low | High | Verify with pilot post; check `.Page.Resources` is non-empty |
| RSS feed absolute image URLs break | Medium | Low | After migration, check RSS output for image URLs; aliases don't fix RSS |
| `attractor` post (5 images) has missing or misnamed WebPs | Low | Medium | Audit `static/images/posts/` before moving attractor's files |
| Posts without images break during migration | Low | Low | Dual-support handles missing `image` param gracefully (existing behavior unchanged) |
| RSS feed image URLs break | Medium | Medium | Verify `public/index.xml` image URLs point to bundle paths; aliases don't fix RSS |
| Pagefind indexes alias redirect pages | Medium | Medium | Hugo alias pages include `noindex` by default; spot-check search results after migration |
| Intermediate section pages appear in sitemap/search | Medium | Low | Suppress with `_index.md` cascade config in Phase 0 before migration |
| `.Page.Resources.GetMatch` doesn't populate as expected | Low | High | Validate with test bundle in Phase 0 before writing any theme templates |

## Security Considerations

- No new inputs, APIs, or trust boundaries
- `aliases` generate static HTML with `<meta http-equiv="refresh">` — no JS, no injection
- Image paths move but remain static files; no change to serving behavior
- No new dependencies
